  • Les classes et les attributs

    • .balises Dans html.html. S'applique à un ul. Définit une liste de balises et leur définitions.
      Emmet: ul.balises>li[data-n=0]>(dfn>code)+.definition+(.exemple>iframe.code.no-nos[data-lang=html]>p)
    • .bloccodeDéfinit un bloc de code. Chacun de ses div est nowrap. Utilisé pour définir une syntaxe (pas un exemple).
    • .bodyLe corps de la page (ou d'une section)
    • .codeÀ mettre dans un iframe indique à Prism et au javascript de le traiter.
    • .colonnesDispose le contenu en colonnes à l'aide de flex.
    • .commandesAppliqué à un ul. Définit une liste de commandes. Noeamment en javascript.
    • .definitionIdentifie la définition d'un terme précédemment nommé à l'aide de dfn.
    • .definitionsAppliqué à un ul. Définit une liste de définitions (voir .balises)
    • .exempleIdentifie un exemple. Contient souvent un iframe qui est géré par programmation. Voir plus bas.
      Emmet: .exemple>iframe.code.no-nos[data-lang=html]>p
    • .exemplesUne balise contenant de petits exemples non géré par Prism.
    • .foldedAjouté par programmation à un article lorsque l'usager clique sur le h2
    • .glossaireAjouté à un ul. Définit un glossaire. Peut-être inutile.
    • .interfaceConteneur de la page
    • .keywordIdentifie un mot-clé du langage
    • .language-javascriptDéfinie par Prism. À mettre dans la balise code de la pair pre>code.
    • .no-nosIndique de ne pas mettre les numéros de ligne. Est géré par Javascript. À mettre dans la balise pre de la pair pre>code
    • .parametresIdentifie les paramètres d'une fonction. Permet le mode concis.
    • .prefixeIdentifie le préfix (objet parent) d'une propriété ou d'une méthode. Permet le mode concis.
    • .sectionsle contenu sera divisé en colonnes à l'aide du css columns. (À revoir)
    • .termeDéfinit le prototype d'une propriété ou d'une méthode.
    • [data-n]Indique le niveau de difficulté/importance du concept. Est habituellement appliqué à un li ou un article. Les valeurs possibles sont : 0, 1, 2, (3)
    • [data-lang]Est appliqué à un iframe. Indique par programmation à Prism quel langage utiliser pour la syntaxe.

  • Les exemples (iframe)

    Pour ajouter un exemple dans une page, on n'a pas besoin de transformer les codes en entités. On n'a qu'à mettre les codes dans le contenu d'un iframe (sans src) et, par programmation, le code sera utilisé par Prism.

    Emmet : div.exemple>iframe.no-nos[data-lang=html]>p

    On peut également récupérer le code provenant d'un dépôt Github. On indique alors la fourchette de lignes à utiliser.

    Emmet : div.exemple>a[href='URL#L01L99'>{texte du lien}

    On remplace URL par l'adresse absolue du fichier raw dans github. On indique le début et la fin du code à ;la place de L01L99. Github va également surligner les lignes lorsque l'on clique sur le lien.
